说句得罪人的话：明年才用上 Claude Code 的人，和今年就已经在用cc的人，已经不在一个世界了。

## Anthropic 内部已经变了

Anthropic 官方透露：在 Anthropic 内部，大部分代码现在由 Claude Code 编写，工程师的角色已经转向架构设计、产品思维和持续编排——管理多个 AI Agent 并行工作，给方向，做决策。

这不是什么遥远的趋势预测。

**这是正在发生的事。**

我写了一个开源项目叫 Media Downloader，全平台媒体下载工具，支持 Twitter/X、Instagram、TikTok、YouTube、Bilibili、Reddit 六个平台。

全程 Claude Code 自己写，这就是编排。你不再是那个写代码的人，你是那个说"我要什么"的人。

## 能理解言外之意的，才叫协作者

能执行指令的工具到处都是。

能理解你言外之意的，才叫协作者。

过去这种能力只属于资深同事——你说"这个不太对"，他能听出来你到底想要什么。

现在一个终端工具也能做到了。

你说这不让人后背发凉？

## 差距不在工具，在范式

这是我真正想聊的。

很多人还在讨论：该学哪个 AI 编程工具——Cursor、Copilot、Claude Code、Codex。好像这件事的核心是工具选择。

**不是的。**

核心是：**你有没有意识到工作方式已经变了。**

当你还在想"这个 CSS 动画怎么写 cubic-bezier"的时候，另一个人正在想"这个页面的视觉叙事应该是什么节奏"。

当你还在逐行调试 i18n 配置文件的时候，另一个人已经截个图说了句"要有中文"，然后去想下一个功能了。

这两种人用的可能是同一个工具。

**但他们活在不同的效率维度里。**

## 差距是指数级的

说句可能得罪人的话：

今年用上 Claude Code 的人，和明年才用上的人之间的差距，就像坐了十年的牢。

不是因为你变差了，是因为外面的人已经用完全不同的速度在迭代。

一个人在一年里用"对话式交付"做了 20 个项目、试了 50 个想法、验证了 10 个方向。

另一个人用传统方式，可能还在第 3 个项目的前端动效上挣扎。

差距不是线性的。是指数级的。

## 你是在用 AI，还是在和 AI 协作？

不想贩卖焦虑。

但我确实觉得，现在这个时间点有一件事值得认真想想——

你是在用 AI，还是在和 AI 协作？

前者是把它当搜索引擎，问一句答一句。

后者是把它当团队成员，给方向、看结果、提反馈、推进度。

两种用法，同一个工具，完全不同的产出。

现在就开始用的人，和等着看的人，最后会站在完全不同的地方。

**这次不是狼来了。**

**狼已经在你旁边坐下了。**
