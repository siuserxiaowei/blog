# Personal Site Design Rules

这份规则从 `ui-ux-pro-max-skill` 的方法论里提炼而来，但只保留适合当前站点的部分。

## 1. Site Identity

- 站点不是“个人简历页”，而是一个公开更新的独立工作台。
- 首页负责说明我在做什么、为什么做、最近在推进什么。
- 工具页负责降低重复劳动，应该更像目录产品，而不是卡片拼贴。
- 文章页负责沉淀判断，视觉上要给内容让路。

## 2. Core Modes

### Home

- 角色：公开工作台。
- 结构：主张 -> 当前重点 -> 证明材料 -> 三个主要入口。
- 页面里每个区块都必须回答一个问题：
  - 我是谁
  - 我在做什么
  - 你下一步该点哪里

### Tools

- 角色：浏览器内工具目录。
- 结构：搜索与筛选 -> 推荐入口 -> 分类目录。
- 不能只按“类别”堆卡片，要同时给“使用场景”和“先从哪里开始”的引导。

### Writing

- 角色：判断日志。
- 重点是可读性、节奏和信息层级，不用过强的装饰特效。

## 3. Visual Direction

- 方向：editorial utility / independent workbench
- 气质：克制、明确、持续更新中的产品感
- 关键词：signal-first, deliberate, structured, browser-native

## 4. Typography

- 标题字体负责建立判断感和密度，不负责“炫技”。
- 正文字体优先可读性，中文和英文要处在同一个系统里。
- Mono 只用于标签、统计、状态、工具元信息。
- 标题和正文的反差来自字号、字重和留白，不靠过多装饰。

## 5. Color Rules

- 整站只保留一主一辅两个强调色。
- 中性色是第一层，强调色只负责引导，不负责铺满页面。
- 避免霓虹、AI 紫粉渐变、彩虹式多重点。
- 卡片边框和背景差异要足够清楚，不能靠模糊特效硬撑层级。

## 6. Layout Rules

- 首页优先用“信息块”而不是“观赏块”。
- 卡片必须有任务：解释、引导、证明、跳转，至少满足一个。
- 工具目录要优先服务扫描和搜索，不要让用户被视觉效果打断。
- 减少无意义的满屏均匀网格，重要信息要有明确主次。

## 7. Motion Rules

- 动效只保留进入、悬停、状态切换三类。
- 悬停位移要轻，默认 1px 到 3px。
- 背景氛围要弱，不能抢正文。
- 如果某个效果不增加可理解性，就删掉。

## 8. Anti-patterns

- 不混搭多个风格语言。
- 不用大面积装饰性渐变模拟“高级感”。
- 不让每张卡片都长得一样且承担一样的任务。
- 不把“分类”误当成“导航”。
- 不为炫而炫；有内容的地方，内容优先。

## 9. Implementation Checklist

- 首页是否一屏内说清楚“我是谁 + 我在做什么 + 先看哪里”
- 工具页是否一屏内完成“搜索 + 推荐 + 信任建立”
- 卡片是否能在 2 秒内扫明白用途
- 英文标签是否只是辅助，而不是抢阅读焦点
- 特效去掉后，页面是否依然成立
